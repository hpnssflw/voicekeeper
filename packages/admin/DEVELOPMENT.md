# Development Guide

## Проверка типов и линтинг

### Проверить все типы сразу (показывает все ошибки)

```bash
npm run type-check
```

Это покажет **все** ошибки TypeScript сразу, а не останавливается на первой.

### Проверка типов в watch режиме

```bash
npm run type-check:watch
```

### Проверка линтера

```bash
npm run lint
```

### Исправить ошибки линтера автоматически

```bash
npm run lint:fix
```

### Полная проверка (типы + линтинг)

```bash
npm run check
```

### Билд с проверкой типов

```bash
npm run build:check
```

Это сначала проверит все типы, и только если ошибок нет - запустит билд.

## Рекомендации

1. **Перед коммитом**: запустите `npm run check` чтобы убедиться, что нет ошибок
2. **Перед билдом**: используйте `npm run build:check` вместо `npm run build`
3. **Во время разработки**: используйте `npm run type-check:watch` в отдельном терминале

## Почему билд падает по одной ошибке?

Next.js использует webpack для сборки, который останавливается на первой ошибке TypeScript. Это стандартное поведение.

Чтобы увидеть **все** ошибки сразу, используйте `npm run type-check` перед билдом.
